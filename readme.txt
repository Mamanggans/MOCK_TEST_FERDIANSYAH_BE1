REST API adalah singkatan dari Representational State Transfer Application Programming Interface. RESTful API bekerja dengan cara memanipulasi resource dan representasi. Representasi ini saling dipertukarkan di antara pengguna dan server melalui antarmuka terstandar dan protokol komunikasi tertentu, biasanya HTTP .

Dalam RESTful API, pengguna mengirimkan permintaan melalui HTTP ke server untuk menggunakan fungsi suatu aplikasi. Server kemudian memproses permintaan tersebut dan mengirimkan respons kembali ke pengguna.

RESTful API memenuhi standar yang ditentukan oleh gaya arsitektur REST. Arsitektur ini menunjukkan karakteristik seperti arsitektur client-server, cacheability, layer system, dan lain-lain.Berikut adalah langkah-langkah cara kerja REST API:

1. HTTP Request dari Klien: Klien membuat HTTP request ke server untuk mengakses atau memanipulasi sumber daya tertentu2.
2. Pengiriman Permintaan ke Endpoint: Permintaan tersebut dikirim ke endpoint yang sesuai2.
3. Proses Permintaan di Server: Server kemudian memproses permintaan tersebut2.
4. Pembuatan Respons: Setelah permintaan diproses, server akan membuat respons2.
5. Pengiriman Respons ke Klien: Respons tersebut kemudian dikirim kembali ke klien2.
6. Pemrosesan Respons di Klien: Klien menerima dan memproses respons tersebut2.
Interaksi Lanjutan (Opsional): Jika diperlukan, klien dapat membuat permintaan tambahan berdasarkan respons yang diterima2.
